# k6 scenarios

Основной учебный маршрут:

1. `01-smoke.js` - проверить, что сервис отвечает.
2. `02-single-endpoint.js` - первая нагрузка на одну ручку.
3. `03-business-flow.js` - простой бизнес-сценарий: поиск и создание заказа.
4. `04-load.js` - обычная ожидаемая нагрузка.
5. `05-stress.js` - постепенно увеличиваем нагрузку.
6. `06-spike.js` - резкий всплеск трафика.
7. `07-soak.js` - длительная стабильная нагрузка.

В основных сценариях `check()` специально простые. Для первого курса достаточно проверять, что endpoint возвращает ожидаемый HTTP status. Более детальные проверки можно добавить позже, когда группа уже понимает VU, duration, thresholds и Grafana Cloud.
